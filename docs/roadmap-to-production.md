# Temu / 店小秘自动上品系统长期开发路线

本文档固定记录项目目标、开发顺序和当前进度。后续持续开发优先按这里推进，除非业务目标变化。

## 理想目标

最终系统应具备以下能力：

- 从 1688、CSV、Excel、手动录入导入商品。
- 自动生成标题、卖点、描述、属性、SKU、价格。
- 通过 Playwright 进入店小秘完成 Temu 草稿填写。
- 自动识别页面字段、页面异常和执行失败原因。
- 人工只负责最终审核和确认发布。
- 每个任务都有日志、截图、失败原因、报告和可重试能力。

## 开发阶段

### 阶段 1：真实页面适配

目标：让店小秘自动填表稳定跑通。

- 页面快照采集：输入框、按钮、SKU 表格、页面文本、截图。
- 店小秘适配器：字段识别、SKU 行匹配、保存按钮定位。
- 基于真实店小秘页面快照持续校准选择器。

完成标准：能稳定把单个商品填入店小秘草稿，不误点发布。

### 阶段 2：任务数据闭环

目标：让任务来自真实商品数据，而不是 mock。

- CSV / Excel 导入商品。
- 手动新增商品，支持多 SKU、属性、图片。
- 已导入任务可编辑，编辑后重建定价和草稿。
- 后端保存商品、任务、执行状态。
- 控制台选择商品并生成刊登草稿。

完成标准：不用改代码，也能在控制台创建、修改真实上品任务。

### 阶段 3：AI 刊登内容生成

目标：自动生成可用的刊登内容。

- 商品标题生成。
- 英文卖点生成。
- 描述生成。
- 类目建议。
- 属性补全。
- SKU 名称规范化。
- 敏感词和侵权风险检查。

完成标准：生成内容可人工小改后直接使用。

### 阶段 4：核价与利润规则

目标：自动算出不亏钱的价格。

- 汇率配置。
- 平台费配置。
- 物流费配置。
- 目标毛利率配置。
- 最低利润保护。
- SKU 单独定价。
- 价格异常提醒。

完成标准：每个 SKU 都有建议售价、底价和利润说明。

### 阶段 5：自动化执行中心

目标：批量执行任务，有状态、有日志、可恢复。

- 任务队列。
- 执行状态：待执行、执行中、已填草稿、失败、待人工确认。
- 每步截图。
- 每步日志。
- 失败自动保存现场。
- 一键重试。
- 批量暂停 / 恢复。

完成标准：可连续处理多条上品任务，失败后能知道原因。

### 阶段 6：人工审核工作台

目标：人在最终发布前高效审核。

- 展示原始商品信息。
- 展示 AI 生成标题、卖点、描述。
- 展示价格和利润。
- 展示店小秘填写截图。
- 标记风险项。
- 审核通过后再允许发布。
- 审核不通过可退回修改。

完成标准：人工审核一个商品控制在 1 分钟内。

### 阶段 7：发布与风控

目标：安全进入准自动发布。

- 默认只保存草稿。
- 发布必须开启独立开关。
- 发布前二次确认。
- 高风险商品禁止自动发布。
- 价格异常禁止自动发布。
- 缺图、缺属性、缺库存禁止自动发布。
- 保留发布截图和结果。

完成标准：可安全小批量半自动发布。

### 阶段 8：批量商品能力

目标：从单品扩展到批量上品。

- 批量导入。
- 批量生成草稿。
- 批量核价。
- 批量填店小秘。
- 批量失败重试。
- 批量导出执行报告。

完成标准：一天可稳定处理几十到几百个商品草稿。

### 阶段 9：质量和稳定性

目标：减少人工维护成本。

- Playwright 自动化回归测试。
- 店小秘页面快照对比。
- 字段识别失败报警。
- 任务数据校验。
- 错误分类。
- 日志查询。
- 配置备份。

完成标准：店小秘页面小改版时，系统能提示哪里坏了。

## 推荐开发顺序

1. 店小秘页面快照采集。
2. 店小秘适配器。
3. 单商品稳定自动填草稿。
4. 执行日志和截图。
5. CSV / Excel 商品导入。
6. 手动录入与已导入任务编辑。
7. 核价规则。
8. AI 内容生成。
9. 控制台任务管理。
10. 批量执行。
11. 审核发布工作流。

当前优先级：真实店小秘页面适配 + 商品数据闭环 + 可审核的草稿工作台。
